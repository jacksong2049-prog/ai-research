# 🔒 安全策略

## 支持的版本

| 版本 | 支持状态 |
|------|---------|
| master (最新) | ✅ 安全更新 |
| 其他版本 | ❌ 不提供安全更新 |

## 报告安全漏洞

我们非常重视安全问题。请通过以下方式报告安全漏洞：

### 📧 邮箱

发送邮件至 **zhangjiayang6835@gmail.com**，标题请以 `[SECURITY]` 开头。

### 🐛 GitHub Issues

**请不要在公开的 Issues 中报告安全漏洞！** 

对于非敏感的安全问题，可以在 Issues 页面选择 "Bug report" 模板提交。

### 预期响应时间

- 确认收到：24小时内
- 初步评估：72小时内
- 修复时间：根据严重程度 3-14 天

### 漏洞披露流程

1. 报告者提交安全漏洞报告
2. 维护者确认并评估严重程度
3. 开发修复方案
4. 发布安全更新
5. 公开披露（修复后）

## 安全最佳实践

使用本项目时，请注意：

- 始终使用最新版本
- 不要在公开环境中暴露敏感配置
- 定期更新依赖：`pip install --upgrade -r requirements.txt`
- 在沙箱环境中运行评测引擎

感谢你帮助我们保持项目的安全性！
